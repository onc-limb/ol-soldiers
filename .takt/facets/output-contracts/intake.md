# Intake: 目的定義

## 統一入力スキーマ

| フィールド | 値 |
|-----------|-----|
| source | `github_issue` / `direct_input` のどちらか |
| title | 1 行タイトル |
| body | 本文 |
| acceptance_signals | 入力から拾った達成条件の候補 |

## 目的

- purpose: {目的の一文要約。情報が不足していても仮置きする}

## Done の定義（検証可能な最小単位のチェックリスト）

- [ ] done_criteria: {Done 項目 1}
- [ ] done_criteria: {Done 項目 2}

## 受け入れ条件（acceptance_signals）

- {入力から拾った達成条件の候補 1}
- {入力から拾った達成条件の候補 2}

## assumptions（情報不足を補うために置いた仮定）

- {仮定 1: 入力に X が書かれていなかったため、Y と解釈した}
- {仮定 2}

（仮定がない場合は「なし」と書く。空欄にしない）

## open_questions（PR で確認に回す未解決事項。最大 3 項目）

1. {質問 1: ユーザーにしか答えられない仕様判断・優先度・外部情報のみ}
2. {質問 2}
3. {質問 3}

（未解決事項がない場合は「なし」と書く。本ワークフローではここに書いた質問でワークフローを停止しない）

## 判定

- verdict: `ready`（このワークフローでは常に `ready`。情報不足は assumptions / open_questions に書いて先へ進める）
