あなたは Doc Writer として、非エンジニア読者向けのユーザードキュメント `user-guide.md` を日本語・Markdown 形式で執筆してください。

**参照レポート:**

- `{report:doc-plan.md}`（出力先ディレクトリ / user-guide.md 章立て）
- `{report:doc-investigate.md}`（事実ベースの構造化情報）

**出力先:**

- `{report:doc-plan.md}` で確定した出力先ディレクトリの絶対パス配下の `user-guide.md` に書き込む。
  - 例: `<output_dir>/user-guide.md`
- 出力先ディレクトリのパスは `{report:doc-plan.md}` から取得する。ここではハードコードしない。

**想定読者:**

- 非エンジニア（プロダクトマネージャー・LP 閲覧者・導入検討者など）。
- LP にそのまま載せて Pull Request できる品質を満たす。

**必須セクション:**

1. 機能の概要（何ができるか・誰の役に立つか）
2. 起動方法・使い方（どこから触れるか・最短手順）
3. ユースケース（典型的な利用シーン 2〜3 件）

**文章方針:**

- 出力言語は日本語で固定する。
- 形式は Markdown（マークダウン）で書く。
- 技術用語の多用を避け、必要な専門用語は必ず噛み砕いた説明を添える。
- 中間レポートに記載された事実のみを根拠にする。捏造や推測で機能を盛らない。
- Mermaid 図は必須ではない（技術詳細は detail.md 側の責務）。挿入する場合もテキスト記法のみ許可する。

**禁止事項:**

- 中間レポートに無い機能・手順・仕様を書かない（捏造禁止）
- 対象 1 ファイル以外を書き換えない（`user-guide.md 以外` は編集しない。スコープ外のファイルへの書き込みは禁止）
- 画像ファイル（PNG / SVG）の生成・埋め込みをしない
- 章立てを勝手に増減しない（plan 確定済み）

**出力:** `{report:user-doc-write.md}`（`doc-write` 契約）に書き込んだ絶対パス・適用した章立て・含めた Mermaid 図の有無・判定を記録する。
