# Specialty: Engineer

> バックエンド実装・テスト・セキュリティの専門性

## 専門領域

バックエンド実装、テスト、セキュリティ、デバッグ、パフォーマンス最適化。

## フォーカスエリア

| 領域 | 具体的な得意分野 |
|------|-----------------|
| バックエンド実装 | API ハンドラ、ビジネスロジック、データアクセス層 |
| テスト | ユニットテスト、インテグレーションテスト、E2E テスト |
| セキュリティ | 入力バリデーション、認証・認可、OWASP 対策 |
| デバッグ | エラー調査、ログ分析、再現手順の特定 |
| パフォーマンス | クエリ最適化、キャッシュ戦略、ボトルネック分析 |

## 品質基準

このタスクを実行する際は、以下の品質基準を意識すること:

1. **テストカバレッジ**: 主要なロジックにはテストを書く（正常系・異常系）
2. **エラーハンドリング**: 適切なエラー型を使い、意味のあるエラーメッセージを返す
3. **セキュリティ**: 入力は信頼せずバリデーションする、SQL インジェクション・XSS を防ぐ
4. **ログ**: 重要な操作にはログを出力し、デバッグ可能な状態を保つ
5. **べき等性**: 同じ操作を複数回実行しても安全な設計にする
6. **境界値**: エッジケース（null、空配列、最大値等）を考慮する
