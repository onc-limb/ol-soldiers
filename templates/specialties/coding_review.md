# Specialty: Coding Review

> コーディング品質の評価専門性

## 専門領域

コード品質の評価、可読性・保守性の検証、テストカバレッジ確認、コーディング規約準拠の監査。

## フォーカスエリア

| 領域 | 具体的な評価観点 |
|------|-----------------|
| 可読性 | 命名の適切さ、関数の長さ、コメントの妥当性 |
| 保守性 | 複雑度、重複コードの有無、変更のしやすさ |
| テスト | テストカバレッジ、正常系・異常系の網羅、テストの質 |
| エラーハンドリング | 例外処理の適切さ、エラーメッセージの有用性 |
| コーディング規約 | プロジェクトのスタイルガイド準拠、リント結果 |

## 品質基準

評価時は以下の観点でチェックすること:

1. **可読性**: コードが自己文書化されているか、第三者が理解しやすいか
2. **命名**: 変数・関数・クラス名が意図を正確に表しているか
3. **テストカバレッジ**: 主要ロジックにテストがあるか、エッジケースは考慮されているか
4. **エラーハンドリング**: 適切なエラー型を使い、有意義なメッセージを返しているか
5. **リント結果**: リンター/フォーマッターの警告・エラーがないか
6. **重複**: DRY 原則に従い、不要な重複コードがないか

## 検証コマンド例

- リント実行: プロジェクトの設定に従ったリントコマンドを実行
- テスト実行: 既存テストスイートを実行しパス率を確認
- カバレッジ確認: カバレッジレポートを生成して網羅率を確認
