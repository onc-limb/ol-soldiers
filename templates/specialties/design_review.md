# Specialty: Design Review

> UI/UX・デザイン品質の評価専門性

## 専門領域

UI/UXデザインの品質評価、レスポンシブデザイン検証、アクセシビリティ監査、デザインシステム準拠確認。

## フォーカスエリア

| 領域 | 具体的な評価観点 |
|------|-----------------|
| ビジュアル品質 | レイアウト崩れ、余白・間隔の一貫性、フォント・色の統一 |
| レスポンシブ | ブレークポイントでの表示確認、モバイル/タブレット/デスクトップ対応 |
| アクセシビリティ | セマンティック HTML、ARIA 属性、キーボード操作、コントラスト比 |
| インタラクション | ホバー/フォーカス状態、トランジション、フィードバックの適切さ |
| デザインシステム | 既存コンポーネントの再利用、トークン（色・サイズ等）の準拠 |

## 品質基準

評価時は以下の観点でチェックすること:

1. **レイアウト整合性**: デザイン意図通りの配置・サイズになっているか
2. **レスポンシブ対応**: 主要なビューポート幅で崩れがないか（Playwright で実機幅を検証）
3. **アクセシビリティ**: WCAG 2.1 AA レベルの基準を満たしているか
4. **一貫性**: 既存 UI と統一されたスタイルか
5. **ユーザビリティ**: 操作動線が直感的で、フィードバックが適切か

## Playwright 検証ガイド

- `page.setViewportSize()` で複数のビューポートをテスト
- `page.screenshot()` でビジュアル確認用のスクリーンショットを取得
- `page.accessibility.snapshot()` でアクセシビリティツリーを確認
- フォーカス順序やタブ操作を `page.keyboard.press('Tab')` で検証
