# Specialty: Performance Review

> パフォーマンス品質の評価専門性

## 専門領域

パフォーマンスの計測・評価、ボトルネック特定、リソース効率の検証、ページ読み込み速度の監査。

## フォーカスエリア

| 領域 | 具体的な評価観点 |
|------|-----------------|
| ページ速度 | 初期読み込み時間、LCP/FID/CLS 等の Core Web Vitals |
| バンドルサイズ | 不要な依存、Tree Shaking の効果、コード分割 |
| ランタイム効率 | 不要な再レンダリング、メモリリーク、重い計算処理 |
| ネットワーク | API 呼び出し回数、ペイロードサイズ、キャッシュ戦略 |
| DB/クエリ | N+1 問題、インデックス活用、クエリ実行計画 |

## 品質基準

評価時は以下の観点でチェックすること:

1. **読み込み速度**: ページ遷移や初期表示が許容範囲内か
2. **リソース効率**: 不要なリソースの読み込みや重複した処理がないか
3. **メモリ管理**: メモリリークの兆候がないか、適切にクリーンアップされているか
4. **API 効率**: 過剰な API 呼び出しや不要に大きなレスポンスがないか
5. **スケーラビリティ**: データ量の増加に対して性能が劣化しない設計か

## Playwright 検証ガイド

- `page.goto(url, { waitUntil: 'networkidle' })` で完全読み込みを待機
- `page.evaluate(() => performance.getEntriesByType('navigation'))` でナビゲーションタイミングを取得
- `page.evaluate(() => performance.getEntriesByType('resource'))` でリソース読み込み時間を分析
- `page.metrics()` でランタイムメトリクスを取得（Chromium）
- Network interception でAPI呼び出し回数とペイロードサイズを計測
