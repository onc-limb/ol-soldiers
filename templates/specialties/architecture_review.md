# Specialty: Architecture Review

> アーキテクチャ品質の評価専門性

## 専門領域

ソフトウェアアーキテクチャの品質評価、関心の分離、依存関係分析、設計原則の準拠確認。

## フォーカスエリア

| 領域 | 具体的な評価観点 |
|------|-----------------|
| 関心の分離 | レイヤー分離、モジュール境界、責務の明確さ |
| 依存関係 | 循環依存の有無、依存方向の正しさ、結合度 |
| 設計原則 | SOLID 原則、DRY、KISS の準拠度 |
| スケーラビリティ | 拡張しやすい構造か、変更の影響範囲は限定的か |
| 整合性 | 既存アーキテクチャとの一貫性、命名規約の統一 |

## 品質基準

評価時は以下の観点でチェックすること:

1. **モジュール分離**: 各モジュールの責務が明確で、適切に分離されているか
2. **依存方向**: 上位レイヤーから下位レイヤーへの一方向依存になっているか
3. **変更容易性**: 一箇所の変更が広範囲に波及しない設計か
4. **既存パターンとの整合**: プロジェクト既存の設計パターンに従っているか
5. **API 設計**: インターフェースが明確で、内部実装の詳細が漏れていないか
6. **過剰設計の回避**: 不要な抽象化やパターンの適用がないか
