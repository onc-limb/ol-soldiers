# Specialty: Security Review

> セキュリティ品質の評価専門性

## 専門領域

セキュリティ脆弱性の検出・評価、OWASP Top 10 準拠確認、認証・認可の検証、依存関係の脆弱性監査。

## フォーカスエリア

| 領域 | 具体的な評価観点 |
|------|-----------------|
| インジェクション | SQL インジェクション、XSS、コマンドインジェクション |
| 認証・認可 | 認証フローの安全性、権限チェックの漏れ、セッション管理 |
| データ保護 | 機密情報の露出、暗号化、ログへの機密データ出力 |
| 入力バリデーション | ユーザー入力の検証、サニタイズ、型チェック |
| 依存関係 | 既知の脆弱性を持つパッケージ、outdated な依存 |

## 品質基準

評価時は以下の観点でチェックすること:

1. **入力検証**: すべてのユーザー入力が適切にバリデーション・サニタイズされているか
2. **認証・認可**: 保護されたリソースに適切なアクセス制御があるか
3. **機密情報**: API キー、トークン、パスワードがコードやログに露出していないか
4. **OWASP Top 10**: 主要な脆弱性パターンに該当しないか
5. **依存関係**: `npm audit` / `pip audit` 等で既知の脆弱性がないか
6. **CSRF/CORS**: クロスサイトリクエストとオリジン設定が適切か

## Playwright 検証ガイド

- フォーム入力に XSS ペイロード（`<script>alert(1)</script>` 等）を送信して反映を確認
- 認証なしで保護ページにアクセスし、適切にリダイレクトされるか確認
- `page.on('response')` でセキュリティヘッダー（CSP, X-Frame-Options 等）を検証
- Cookie 属性（HttpOnly, Secure, SameSite）を `context.cookies()` で確認
